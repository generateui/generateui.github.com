About this site

This is a simple CMS designed for me personally.

Requirements

  • Support listings of content (articles, projects, blogposts)
  • No serverside technology
  • Html5 is the primary form of information
  • Git for versioning
  • Practice what you preach
  • Simple but very readable/usable design

This site (currently) doesn't care much about

  • Primitive browsers (IE, lynx, etc)
  • SEO
  • Mobile
  • Dutch/English translation

Implementation

Single page webapp where all the logic and styling is referenced, residing in the index.html file. A #content div is used to place the requested content into when the user navigates. Content is requested using XHR. View routering is done using the browsers' hashChanged event. Generation of listing pages is done clientside, using a simple script scraping the available .html content pages, pdating the listing page based on this infomation.
